####  不建议用*{margin: 0px;padding: 0px;}这种写法，尤其大企业，影响效率！ //个人网站或小网站可以这么写
#####  详细讲解
```
1.自己学习搭建网站时，有这么一句CSS
<head>

	<style type="text/css">
		*{margin: 0px;padding: 0px;}

	</style>
</head>

【注】：*{margin: 0px;padding: 0px;}
*表示所有元素。*叫“通配符”,用来匹配页面上所有元素。
为什么要加*{margin:0; padding:0;}呢？因为body ,ul, li ,p,h1~h6,dd,dt 等……都有默认的margin 或padding值的，
这样就不方便我们精确控制，加上这句就可以删除浏览器这些默认值，方面日后的设置和维护。
所以我们干脆就先把所有元素的margin、padding先设置为0，以后根据需要再单独设置某一项的margin和padding数值 。

2.但有的前端大神和百度、新浪、腾讯等设计时，会用第二种思路：他们习惯一般把 body 和ul li 等的边距重置成0
即：
<head>

	<style type="text/css">
		body,ul,li{margin:0px;padding:0px;}
        
	</style>
</head>
注：body边距重置0是为了防止以后body外面留白，比如有导航栏时候。
    ul，li边距重置为0是因为每个li列前默认有小圆点，很多时候不需要小圆点，去掉后也有默认的边距。
    这里提醒下：
        *{margin:0px;padding:0px;}  *是代表网页包括body在内的所有元素,表示任意元素的外边距和内这距都是0px;
        body{margin:0px;padding:0px;} 只表示body这个大元素的外边距和内边距是0px（不含body体内部元素）。
                                      body体外部默认有8px的margin边距

为什么大企业都用第2种思路呢？
1）效率问题：
*{...} 会把所有标签都匹配一次。而实际上需要把 内、外边距归零 的玩意没那么多，用 * 就更耗资源。
一般页面感觉不出来，解析时间用 * 会多花百分几秒，但复杂的页面可能多花的就是十分之几秒了。新浪、百度这些大站很在乎这个的，
这里零点零几秒，那里零点零几秒加起来就是半秒一秒。新浪、百度是对的，大企业花钱养那么大个前端团队专门干这些，不是吃白饭的。
下面是百度主页面（www.baidu.com）的css对于{margin:0px;padding:0px;}的设计，可以看到用的是写法2
```

```
百度新浪腾讯有的页面是把所有的标签并在一起 body,h1,h2,h3,ul,li,input,div,span,a,form …… { margin:0 ; padding:0 }

2）*写法显得太粗暴不专业：
大型网站不推荐用*，因为例外的规则需要重新写麻烦。
因为我们用到的标签只是少数几个，没必要把所有标签都重置，*写法会显得不专业！--这是w3c里面说的

【总结】
小型网站：如个人博客，可以用写法1。因为网站规模小，标签内容少，所以* 就可以重置一些基本样式，浏览器没什么大错位就满足了。
大型网站：用法2。

[拓展探索]
据说百度在移动端搜索页已经在用 *{ margin:0 ; padding:0 }
很多人都说*效率低、慢，但据说有人做过实验，这两种写法资源占用一样，加载速度一样？
上面值得去探索。未有结论前，建议用写法2。
```
